# Security Policy

Thank you for helping keep our project and users safe. We welcome security research and responsibly disclosed vulnerability reports.

## Scope and Testing Rules

* ✅ **Allowed:** You are free to conduct non-destructive security tests on our test environment at **[test.dropkey.site](https://test.dropkey.site)**.
* ❌ **Prohibited:** You must **not** test, attack, or otherwise touch our main production host at **[dropkey.site](https://dropkey.site)**.

## Reporting a Vulnerability

If you discover a vulnerability, please adhere to the following disclosure process:

1. **Report Privately:** Email the vulnerability details privately to **levongabrielyan44@gmail.com**. Please do not open public GitHub issues for security flaws.
2. **Allow Time for Remediation:** Give us a reasonable amount of time to investigate and deploy a fix before sharing it with any third parties.
3. **Public Disclosure:** Once we have confirmed the fix is in place, you are welcome to disclose the vulnerability publicly.
